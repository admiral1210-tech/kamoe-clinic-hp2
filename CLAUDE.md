# トークン節約ルール

- 作業が一段落するたびに `/compact` を提案する（目安: 長い調査・複数ファイル編集・検証ループの後）
- ファイルを読む前に Grep/Glob で対象を絞り、不要な全文読み込みを避ける
- preview 検証は CSS/レイアウト変更のみ実施。テキスト修正・設定変更のみの場合はスキップ

# ツール使用の優先順位

- ファイル検索: Glob → Grep → Read の順で絞り込んでから読む
- Agent ツール: 3回以上の Grep/Glob が必要な広域調査のみ使用。単純な検索は直接ツールを使う
- preview_screenshot: 視覚的変更の確認のみ。テキスト・設定変更はスキップ

# セッション管理

- 1セッション = 1機能/1ページ を原則とする
- 長くなったら `/compact` → 完了後は `/clear` で新セッション開始
- 複数ファイルを横断する大規模作業の前に `/compact` を実行

# `/compact` 応答ルール（必須）

ユーザーが `/compact` と送ったとき、短い説明のあと**必ず**次を出力する。

1. **コピペ用ブロック**  
   - 次のフェンスで**要約全文を1ブロックに閉じる**（ユーザーがフェンスごと選択コピーしやすいようにする）。  
   - フェンス直前に1行だけ「下のブロックをそのままコピーして新セッションに貼れます」と書く。  
   - フェンス内は**プレーンテキスト**（先頭に `## セッション要約（コピペ用）` を含む）。言語タグは `text` とする。

2. **フェンス内テンプレート**（空欄はそのセッションの事実で埋める）

```
## セッション要約（コピペ用）

依頼: （ユーザーが何をしたいか1〜3行）

現状: （いまのコード・方針・ブロッカー）

決定・合意: （確定した仕様・デザイン・方針。なければ「なし」）

未完了: （残タスク・未検証）

次: （新セッションで最初にやること）
```

3. 会話のメタ説明（「トークン節約のため」など）は**フェンスの外**に短く書く。
