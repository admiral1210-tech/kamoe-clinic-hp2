# フォームSaaS 3省2ガイドライン適合チェックリスト

> 作成日: 2026-04-13  
> 目的: 問い合わせフォームをGoogleフォームから医療対応SaaSへ移行する際の法令・ガイドライン適合確認  
> 評価対象: **FormRun** / **kintone** / 参考比較: **Formstack / HubSpot Forms**

---

## 前提：当サイトのフォームが収集するデータの分類

| 収集項目 | 個人情報区分 | 根拠 |
|----------|------------|------|
| 氏名 | 個人情報 (一般) | 個人情報保護法 §2 |
| 電話番号 | 個人情報 (一般) | 同上 |
| メールアドレス | 個人情報 (一般) | 同上 |
| 相談内容（自由記述） | **要配慮個人情報になり得る** | 病状・既往歴等を含む場合 |
| 希望訪問日時 | 個人情報 (一般) | — |

**結論**: 相談内容欄は「病状・診断名」を含む可能性があるため、  
**要配慮個人情報（センシティブ情報）**として扱うことが安全。  
これにより**3省2ガイドラインの射程に入る可能性がある**。

---

## 3省2ガイドライン とは

### ガイドライン①（厚生労働省）
**「医療情報システムの安全管理に関するガイドライン 第6.0版」**（2023年5月）  
対象: 医療機関等が運用する情報システム全般

### ガイドライン②（総務省・経済産業省）
**「医療情報を取り扱う情報システム・サービスの提供事業者における安全管理ガイドライン 第1.1版」**（2023年7月）  
対象: クラウドサービス等のシステム提供事業者

> ⚠️ **スコープ判断**  
> 問い合わせフォームが純粋な「連絡先収集」に留まり、診療録・処方箋等を扱わない場合、  
> **②は直接適用されない可能性が高い**。ただし①の「外部委託先管理」条項は依然適用される。  
> 法的確認は専門家（医療情報・個人情報保護の弁護士）に委ねること。

## チェックリスト D: 医療広告・UX要件（クリニック固有）

| # | チェック項目 | 備考 |
|---|------------|------|
| D-1 | フォーム送信時に**個人情報取扱い同意チェックボックス**を設置できる | 医療機関として必須 |

### 判定ロジック

```
フォームが収集するデータ
  └─ 連絡先 + 自由記述（病状含む可能性）
       ├─ 診療録・処方箋等を含まない → 3省2GL②の直接義務なし
       │    → FormRun で十分（個人情報保護法 + 厚労省外部委託管理に対応）
       └─ 将来的に電子カルテ連携を検討 → kintone + ISMAP認証SaaS を選定
```

## 移行アクションプラン

### Phase 1: FormRun 移行（推奨・即時実施可能）

- [ ] FormRun の有償プラン（Starter以上）を契約
- [ ] **個人情報処理委託契約書**を FormRun（Wantedly社）と締結
- [ ] フォーム項目設計:
  - 氏名・電話番号・メールアドレス（必須）
  - 相談内容（任意・自由記述）
  - **個人情報取扱い同意チェックボックス**（必須）
  - reCAPTCHA v3 有効化
- [ ] 自動返信メールテンプレート作成
- [ ] 管理画面MFA設定
- [ ] プライバシーポリシーに「FormRunへの委託」を追記
- [ ] 旧Googleフォームのリンクを新URLに差し替え（全ページ）

### Phase 2: 法務確認（移行後30日以内）

- [ ] チェックリストB・C の△項目について FormRun に書面照会
- [ ] 医療情報・個人情報専門の弁護士によるレビュー依頼
- [ ] レビュー結果をプライバシーポリシーに反映

### Phase 3: 将来対応（必要に応じて）

- [ ] kintone導入：問い合わせ管理を CRM 的に運用したい場合
- [ ] ISMAP登録済みSaaS検討：電子カルテ・医療情報と統合する場合
